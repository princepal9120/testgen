Documentation

TestGen is a multi-language AI test generation tool with one shared engine behind every surface: the interactive TUI, direct CLI commands, repo-local agent wrappers, and an experimental MCP server.

The safest default workflow is review-first: generate dry-run output, inspect structured artifacts and patch operations, then write files only when you are ready.

Installation

Install TestGen locally as a binary, or install from source using the Go module path.

curl -fsSL https://raw.githubusercontent.com/princepal9120/testgen/main/install.sh | bash
irm https://raw.githubusercontent.com/princepal9120/testgen/main/install.ps1 | iex
go install github.com/princepal9120/testgen-cli@latest
Compatibility note:
- GitHub repo: github.com/princepal9120/testgen
- Go module:  github.com/princepal9120/testgen-cli

Configuration

Set one provider API key before generating tests. TestGen currently supports Anthropic, OpenAI, Gemini, and Groq.

# Choose one provider
export ANTHROPIC_API_KEY="sk-ant-..."
export OPENAI_API_KEY="sk-..."
export GEMINI_API_KEY="AIza..."
export GROQ_API_KEY="gsk_..."
  • Anthropic Claude: best default quality for many repos
  • OpenAI: popular general-purpose option
  • Gemini: useful when you want a Google-hosted option
  • Groq: good when low latency matters

Quick Start

Start with analysis, then preview generation, then write files only when you want validation feedback.

1. Inspect a codebase

testgen analyze --path=./examples --cost-estimate --recursive

2. Preview generated tests without writing

testgen generate --file=./examples/python/calculator.py \
  --type=unit \
  --dry-run \
  --emit-patch \
  --output-format json

3. Generate and validate when ready

testgen generate --path=./src --recursive --type=unit --validate

4. Prefer the guided UI?

testgen tui

Usage Modes

People can use TestGen in four practical ways today:

  • TUI: best for first-time human users who want a guided keyboard flow
  • CLI: best for local development, scripting, and CI automation
  • Agent wrappers: best when Codex, Claude Code, or OpenCode should inspect dry-run artifacts first
  • MCP: best when your client prefers tool calls over shell execution

Agent Integrations

TestGen ships repo-local integration surfaces that all call the same app layer underneath.

  • Codex / oh-my-codex: .codex/skills/testgen/SKILL.md
  • Claude Code: .claude/commands/testgen.md
  • OpenCode: .opencode/commands/testgen.md
  • MCP: testgen mcp exposes testgen_generate, testgen_analyze, and testgen_validate

Recommended safe mode for agents

testgen generate --file=./src/utils.py --type=unit --dry-run --emit-patch --output-format json

Install wrappers into another repo

./scripts/install-agent-integrations.sh /path/to/target-repo copy

Print an MCP config snippet

./scripts/print-mcp-config.sh testgen

Core Commands

Generate tests

testgen generate --path=./src --recursive --type=unit,edge-cases

# Important flags:
#   --dry-run
#   --emit-patch
#   --validate
#   --output-format json
#   --parallel 4
#   --interactive

Analyze cost and complexity

testgen analyze --path=./src --cost-estimate --detail=per-file --output-format json

Validate existing tests and coverage

testgen validate --path=./src --min-coverage=80 --fail-on-missing-tests

Run as an MCP server

testgen mcp

Supported Languages

TestGen currently detects and generates tests for these languages:

  • JavaScript / TypeScript: Jest, Vitest, Mocha
  • Python: pytest, unittest
  • Go: testing, testify
  • Rust: cargo test
  • Java: JUnit 5-oriented generation via the Java adapter
